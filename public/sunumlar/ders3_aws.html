ECR — Elastic Container Registry

AWS'nin özel image deposudur. Kodunuz güvende kalır, public Docker Hub'a gitmez.

ECR'ye Push

# 1. Repository oluştur
aws ecr create-repository \
  --repository-name menu-classifier \
  --region eu-central-1

# 2. Login
aws ecr get-login-password \
  --region eu-central-1 | \
  docker login --username AWS \
  --password-stdin \
  $ACCOUNT.dkr.ecr.eu-central-1.amazonaws.com

# 3. Tag & Push
docker tag menu-classifier:latest \
  $ACCOUNT.dkr.ecr.eu-central-1.amazonaws.com/menu-classifier:latest

docker push \
  $ACCOUNT.dkr.ecr.eu-central-1.amazonaws.com/menu-classifier:latest

ECS + Fargate

ECS Kavramları

  • Cluster — Container'ların çalışacağı ortam
  • Task Definition — Container ayarları (image, CPU, RAM, port)
  • Task — Task Definition'dan çalışan örnek
  • Service — Kaç Task çalışmalı? (desired count)

Neden Fargate?

  • EC2 sunucu yönetmek zorunda değilsin
  • Patch, update, scale → AWS halleder
  • Kullandığın kadar öde (vCPU + RAM)
  • Demo için ideal: hızlı kur, hızlı sil

CLI ile ECR Push

## Değişkenleri ayarla
REGION="eu-central-1"
ACCOUNT_ID="$(aws sts get-caller-identity --query Account --output text)"
REPO="menu-classifier"
TAG="latest"
ECR_URI="$ACCOUNT_ID.dkr.ecr.$REGION.amazonaws.com"

## ECR'ye login
aws ecr get-login-password --region $REGION | \
  docker login --username AWS --password-stdin $ECR_URI

## Tag & push
docker tag $REPO:$TAG $ECR_URI/$REPO:$TAG
docker push $ECR_URI/$REPO:$TAG

## Doğrula
aws ecr list-images --repository-name $REPO --region $REGION
Push başarılıysa AWS Console → ECR → Repositories'de görünür. ECS artık bu URL'den image'ı çekebilir.

AWS CLI ile Task Definition

aws ecs register-task-definition \
  --family menu-classifier \
  --network-mode awsvpc \
  --requires-compatibilities FARGATE \
  --cpu "1024" \
  --memory "2048" \
  --container-definitions '[{
    "name": "menu-classifier",
    "image": "'$ECR_URI'/menu-classifier:latest",
    "portMappings": [
      {"containerPort": 8080, "protocol": "tcp"}
    ],
    "environment": [
      {"name":"MODEL_PATH","value":"/app/menu_mobilenet_model.h5"},
      {"name":"AWS_REGION","value":"eu-central-1"},
      {"name":"S3_BUCKET","value":"mapindata-menu-images"}
    ]
  }]'

AWS'de Ölçekleme

2. derste sormuştuk: 500 kişi gelince ne olur?
AWS'de cevap tek komut: 
# 5 container'a ölçekle
aws ecs update-service \
  --cluster menu-classifier-cluster \
  --service menu-classifier-service \
  --desired-count 5

# Maliyet kontrolü için 2'ye düşür
aws ecs update-service \
  --cluster menu-classifier-cluster \
  --service menu-classifier-service \
  --desired-count 2

Auto Scaling (Otomatik)

CPU %70'i geçince otomatik ölçekle:

  • Min: 1 task, Max: 10 task
  • CPU > %70 → +1 task ekle
  • CPU < %30 → -1 task çıkar
  • CloudWatch alarmları tetikler

Lokal vs AWS

Hata toleransıMulti-AZ, auto-restart
ErişimDünya geneli
